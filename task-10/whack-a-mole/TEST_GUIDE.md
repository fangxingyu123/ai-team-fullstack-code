# 🧪 打地鼠游戏测试指南

## 快速测试步骤

### 1. 打开游戏
```bash
# 方法 1：直接打开
cd /home/node/.openclaw/workspace/whack-a-mole
# macOS: open index.html
# Windows: start index.html
# Linux: xdg-open index.html

# 方法 2：本地服务器（推荐）
python3 -m http.server 8000
# 访问 http://localhost:8000
```

### 2. 打开浏览器控制台
- 按 `F12` 或右键 → 检查
- 切换到 "Console" 标签
- 清空之前的日志

### 3. 开始测试

#### 测试场景 1：正常游戏流程
1. 点击 "开始游戏" 按钮
2. 观察控制台日志：
   ```
   [SPAWN] 地鼠生成系统启动（事件驱动模式）
   [SPAWN] 开始生成地鼠，currentMoleIndex=-1
   [SPAWN] 新地鼠位置：X, currentMoleIndex 已更新
   [SHOW] 地鼠 X 冒出 (停留 1500ms), currentMoleIndex=X
   ```
3. 点击出现的地鼠
4. 观察控制台日志：
   ```
   [CLICK] 点击洞 X, currentMoleIndex=X, isPlaying=true, score=0
   [CLICK] ✓ 命中！洞 X
   [HIT] 得分计算：10 × 1 = 10, 分数：0 → 10
   🎯 命中！+10 分，连击 x1，总分：10
   ```
5. 继续游戏，观察分数是否正常累加

#### 测试场景 2：快速连击测试
1. 开始游戏后快速连续点击地鼠
2. 观察连击倍数是否正确增加：
   ```
   [HIT] 连击！combo: 1 → 2
   [HIT] 得分计算：10 × 2 = 20, 分数：10 → 30
   [HIT] 连击！combo: 2 → 3
   [HIT] 得分计算：10 × 3 = 30, 分数：30 → 60
   ```
3. 连击达到 5 倍后应该不再增加（设计如此）

#### 测试场景 3：慢速点击测试
1. 点击一次地鼠后等待 3 秒以上
2. 再次点击地鼠
3. 观察连击是否重置：
   ```
   [HIT] 连击中断，重置为 x1 (上次命中：3000ms 前)
   [HIT] 得分计算：10 × 1 = 10, 分数：50 → 60
   ```

#### 测试场景 4：游戏后期测试（关键！）
1. 正常玩游戏到 20-25 秒
2. 分数达到 50+ 后继续点击
3. **关键观察**：每次点击是否还能加分
4. 如果发现问题，记录控制台日志

#### 测试场景 5：边界情况测试
1. 游戏结束前最后一秒点击地鼠
2. 观察是否正确处理：
   ```
   [CLICK] 点击洞 X, currentMoleIndex=X, isPlaying=true, score=100
   [CLICK] ✓ 命中！洞 X
   [HIT] 得分计算：10 × N = P, 分数：100 → 100+P
   ```
3. 时间到后点击应该被忽略：
   ```
   [CLICK] 点击洞 X, currentMoleIndex=X, isPlaying=false, score=150
   [CLICK] 忽略：游戏未进行
   ```

### 4. 问题排查

#### 问题 A：点击地鼠但不加分
**检查控制台日志：**
- 如果看到 `[CLICK] ✗ 未命中！洞 X (地鼠在 Y)`：
  - 说明 `currentMoleIndex` 状态不同步
  - 记录完整的日志序列以便分析

- 如果看到 `[HIT] 忽略：索引不匹配`：
  - 说明点击时地鼠索引已变化
  - 可能是定时器链问题

- 如果看到 `[HIT] 忽略：游戏已结束`：
  - 说明游戏状态异常
  - 检查是否意外触发了结束条件

#### 问题 B：地鼠不再出现
**检查控制台日志：**
- 如果看到 `[SPAWN] 忽略：游戏已结束`：
  - 说明 `gameState.isPlaying` 被设置为 false
  - 检查是否意外触发了结束条件

- 如果没有 `[SPAWN]` 日志：
  - 说明定时器链断裂
  - 记录最后一次地鼠出现后的所有日志

#### 问题 C：连击不增加
**检查控制台日志：**
- 如果看到 `[HIT] 连击中断，重置为 x1`：
  - 说明两次点击间隔超过 2 秒
  - 这是正常行为

- 如果连击始终是 x1：
  - 检查 `gameState.lastHitTime` 是否被正确更新
  - 记录完整的日志序列

### 5. 测试完成检查清单

- [ ] 开始游戏后地鼠正常出现
- [ ] 点击地鼠能正常加分（+10 分）
- [ ] 快速连击时连击倍数递增（最高 x5）
- [ ] 慢速点击时连击重置为 x1
- [ ] 游戏后期（20-30 秒）仍能正常加分
- [ ] 游戏结束后点击被正确忽略
- [ ] 最终分数显示正确
- [ ] 控制台日志清晰可读
