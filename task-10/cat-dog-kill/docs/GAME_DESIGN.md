# 猫狗杀 - 游戏设计文档

## 1. 游戏概述

**游戏名称**: 猫狗杀 (Cat Dog Kill)  
**游戏类型**: 多人联机社交推理游戏  
**目标平台**: iOS (iPhone/iPad)  
**目标用户**: 12+ 岁，喜欢社交推理游戏的玩家  
**游戏时长**: 每局 10-20 分钟

---

## 2. 核心玩法

### 2.1 游戏模式

```
┌─────────────────────────────────────────────────────────┐
│                     游戏流程                            │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  大厅 → 匹配/创建房间 → 角色分配 → 游戏阶段            │
│                                      ↓                  │
│  游戏结束 ← 胜利判定 ← 投票淘汰 ← 紧急会议            │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 2.2 角色系统

#### 🐱 猫咪 (好人阵营)
- **人数**: 2-8 人 (根据总人数调整)
- **目标**: 完成所有任务 OR 找出所有狗狗
- **能力**:
  - 完成任务获取进度
  - 发起紧急会议
  - 参与投票
  - 报告发现的"尸体"

#### 🐶 狗狗 (坏人阵营)
- **人数**: 1-3 人
- **目标**: 淘汰足够多的猫咪 (狗狗数 ≥ 猫咪数)
- **能力**:
  - 破坏设施 (锁门、关灯等)
  - 假装完成任务
  - 参与投票
  - 识别其他狗狗身份

#### 🦊 狐狸 (中立阵营)
- **人数**: 0-2 人 (可选)
- **目标**: 存活到最后 (场上只剩自己)
- **能力**:
  - 隐藏身份
  - 特殊技能 (待定)

### 2.3 胜利条件

| 阵营 | 胜利条件 |
|------|----------|
| 猫咪 | 完成所有任务 OR 投票淘汰所有狗狗 |
| 狗狗 | 存活狗狗数量 ≥ 存活猫咪数量 |
| 狐狸 | 场上只剩狐狸一人 |

- 梳理毛发

## 6. 会议与投票

### 6.1 会议触发

- **紧急按钮**: 任何存活玩家可发起 (每局限 1 次)
- **发现尸体**: 报告淘汰玩家
- **自动触发**: 破坏达到阈值

### 6.2 会议流程

```
1. 所有玩家传送至会议点
2. 显示被淘汰玩家信息
3. 自由讨论 (60 秒)
4. 投票阶段 (30 秒)
5. 公布结果
6. 执行淘汰/跳过
```

### 6.3 投票 UI


## 9. 技术实现

### 9.1 网络同步

```
客户端 ←→ Socket.IO ←→ 服务器
   ↓                        ↓
本地预测              权威状态
   ↓                        ↓
插值平滑              广播更新
```

### 9.2 状态同步频率

| 数据类型 | 频率 | 说明 |
|----------|------|------|
| 玩家位置 | 10Hz | 移动同步 |
| 游戏状态 | 1Hz | 任务/投票等 |
| 聊天消息 | 实时 | 文字聊天 |
| 语音数据 | 20Hz | 语音聊天 (二期) |

### 9.3 断线重连

1. 检测连接断开
2. 自动重连 (最多 3 次)
3. 同步最新游戏状态
4. 恢复玩家位置和操作
