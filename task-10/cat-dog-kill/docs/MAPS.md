# 🗺️ 猫狗杀 - 地图设计文档

## 地图系统概述

游戏支持多张地图，每张地图有独特的布局、任务点和视觉主题。

### 地图数据结构

```typescript
interface GameMap {
  id: string;              // 地图唯一标识
  name: string;            // 地图名称
  description: string;     // 地图描述
  theme: MapTheme;         // 视觉主题
  size: { width: number; height: number };  // 地图尺寸 (100x100 坐标系)
  zones: MapZone[];        // 功能区域
  tasks: TaskLocation[];   // 任务点分布
  obstacles: Obstacle[];   // 障碍物
  meetingPoint: { x: number; y: number };  // 会议集合点
  sabotagePoints: SabotagePoint[];  // 破坏点
}
```

## 🔧 破坏点设计

每张地图有 3-4 个可破坏点：

| 破坏类型 | 效果 | 冷却时间 |
|----------|------|----------|
| 锁门 | 封锁某个区域 30 秒 | 45 秒 |
| 关灯 | 降低全图视野 30 秒 | 60 秒 |
| 断网 | 禁用紧急会议按钮 30 秒 | 90 秒 |
| 加速 | 狗狗移动速度 +50% 15 秒 | 60 秒 |

### 各地图破坏点位置

**宠物乐园**:
- 大门 (锁门)
- 路灯 (关灯)
- 广播室 (断网)

**喵喵别墅**:
- 前门/后门 (锁门)
- 电闸 (关灯)
- 路由器 (断网)

**动物医院**:
- 隔离门 (锁门)
- 主灯 (关灯)
- 呼叫系统 (断网)

**森林营地**:
- 营地入口 (锁门)
- 篝火 (关灯)
- 信号塔 (断网)

**太空站**:
- 气闸门 (锁门)
- 主电源 (关灯)
- 通讯阵列 (断网)

## 📋 任务分布规则

### 短任务 (5-10 秒)
- 每个区域 1-2 个
- 简单互动操作
- 示例：喂食、整理、记录

### 长任务 (15-30 秒)
- 关键区域 1-2 个
- 多步骤操作
- 示例：手术、清理、维修

### 公共任务 (多人协作)
- 中央区域 1 个
- 需要 2-3 人同时操作
- 示例：搬运、启动装置
